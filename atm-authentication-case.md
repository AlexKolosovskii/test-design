# Test Design: Система авторизации банкомата (State Transition Testing)

## Описание
Тестирование системы авторизации на основе диаграммы переходов состояний.

## Состояния и правила
- Ожидание карты → после вставки карты → Карта вставлена
- Карта вставлена → при правильном PIN → PIN введён
- Карта вставлена → после трёх неверных попыток → Блокировка
- PIN введён → после выбора счёта → Счёт выбран → после транзакции → Карта выдана
- Из состояния Блокировка система переходит к выдаче карты
- Из состояний Карта выдана и Блокировка других переходов нет

## Ключевые тест-сценарии

| № | Сценарий | Ожидаемый результат |
|---|---------|---------------------|
| 1 | Вставить карту → ввести верный PIN → выбрать счёт → выполнить транзакцию | Успешное завершение, карта выдана |
| 2 | Вставить карту → трижды ввести неверный PIN | Система блокируется, карта выдана |
| 3 | Попытка выбрать счёт без ввода PIN | Ошибка, действие запрещено |
| 4 | Попытка выполнить транзакцию без выбора счёта | Ошибка, действие запрещено |
| 5 | Попытка взаимодействия с системой до вставки карты | Ошибка, действие запрещено |

## Результат
- Покрыты все допустимые переходы между состояниями
- Проверены недопустимые действия в каждом состоянии
- Подтверждено корректное завершение сессии (выдача карты)
