# Test Design: Управление доступом к удалению постов

## Описание
Тест-дизайн для системы управления правами на удаление постов с применением таблицы принятия решений (Decision Table).

## Бизнес-правила
Пользователь может удалить пост, если выполняется хотя бы одно из условий:
- Пользователь — админ (может удалять любой пост)
- Пользователь — модератор и пост не защищён
- Пользователь — автор именно этого поста и пост не защищён

Обычный пользователь, не являющийся автором поста, не может удалять посты ни при каких условиях.

## Набор тестов

| № | Роль       | Автор поста | Пост защищён | Может удалить |
|---|------------|--------------|---------------|----------------|
| 1 | Админ      | Любой        | Любой         | Да             |
| 2 | Модератор  | Любой        | Нет           | Да             |
| 3 | Модератор  | Любой        | Да            | Нет            |
| 4 | Обычный    | Да           | Нет           | Да             |
| 5 | Обычный    | Да           | Да            | Нет            |
| 6 | Обычный    | Нет          | Любой         | Нет            |

## Применённая техника
- Таблица принятия решений (Decision Table)
- Явное покрытие всех комбинаций ролей, авторства и защиты
- Минимизация: 6 тестов вместо 12 возможных комбинаций

## Примечание
Состояние «Автор поста» относится именно к текущему посту, а не к общей роли пользователя.
